## 🚀 三步开始使用

### 第一步: 安装依赖

```bash
cd claude-proxy
pip install -r requirements.txt
```

### 第二步: 配置环境

```bash
# 复制配置文件
cp .env.example .env

# 编辑 .env,填写你的 API Key
# Windows: notepad .env
# Linux/Mac: nano .env
```

**必填配置**:
```env
UPSTREAM_API_KEY=sk-your-api-key-here
```

### 第三步: 启动服务

```bash
# Windows
start.bat

# Linux/Mac
chmod +x start.sh
./start.sh

# 或直接运行
python -m proxy
```

## 🔌 连接 CC Switch

### 方式 1: Deep Link (最快)

点击或复制到浏览器:
```
ccswitch://provider/import?name=Claude%20Proxy&apiKey=sk-proxy-001&baseUrl=http://127.0.0.1:8000&model=claude-3-5-sonnet-20241022
```

### 方式 2: UI 手动导入

1. 打开 CC Switch
2. 点击 "Add Provider"
3. 选择 "Custom Provider"
4. 填写:
   - Name: `Claude Proxy`
   - API Key: `sk-proxy-001` (任意)
   - Base URL: `http://127.0.0.1:8000`
   - Model: `claude-3-5-sonnet-20241022`
5. 保存并启用
6. 重启 Claude Code 终端

**详细步骤**: 见 [CC_SWITCH_GUIDE.md](CC_SWITCH_GUIDE.md)

## 💡 核心特性

### 1️⃣ 多 Agent 协作

自动根据任务复杂度分配 Agent:

**简单任务** (< 1000 字符):
```
用户请求 → 单 Agent 处理 → 立即返回
```

**复杂任务** (> 5000 字符 或 有工具调用):
```
用户请求 → 多 Agent 路由
           ├─ 优先级分析
           ├─ 队列管理
           └─ 并发控制
```

**配置**:
```env
ENABLE_MULTI_AGENT=true
MAX_AGENTS=3  # 根据机器配置调整
```

### 2️⃣ 灵活上游

支持任意 OpenAI 兼容 API:

**DeepSeek**:
```env
UPSTREAM_BASE_URL=https://api.deepseek.com/v1
UPSTREAM_MODEL=deepseek-chat
```

**OpenAI**:
```env
UPSTREAM_BASE_URL=https://api.openai.com/v1
UPSTREAM_MODEL=gpt-3.5-turbo
```

**自定义服务**:
```env
UPSTREAM_BASE_URL=https://your-api.com/v1
UPSTREAM_MODEL=your-model
```

### 3️⃣ 流式 + 非流式

**非流式** (一次性返回):
```json
{
  "stream": false
}
```

**流式** (实时输出):
```json
{
  "stream": true
}
```

客户端(如 Claude Code)会自动选择。

### 4️⃣ 工具调用

完整支持 Anthropic Tools:

```json
{
  "tools": [
    {
      "name": "get_weather",
      "description": "获取天气",
      "input_schema": {
        "type": "object",
        "properties": {
          "location": {"type": "string"}
        }
      }
    }
  ]
}
```

自动转换为 OpenAI Function Calling。

## 🐛 常见问题

### 问题 1: 端口被占用

**现象**:
```
OSError: [Errno 98] Address already in use
```

**解决**:
```bash
# 方法 1: 修改端口
# 编辑 .env: PROXY_PORT=8001

# 方法 2: 关闭占用进程
# Windows
netstat -ano | findstr :8000
taskkill /PID <PID> /F

# Linux/Mac
lsof -ti:8000 | xargs kill -9
```

### 问题 2: 上游连接失败

**现象**:
```
httpx.ConnectError: Connection refused
```

**检查**:
1. API Key 是否正确
2. Base URL 是否正确 (注意 `/v1` 后缀)
3. 网络是否畅通
4. 是否需要配置代理

**解决**:
```env
# 添加代理(如果需要)
HTTP_PROXY=http://127.0.0.1:7890
HTTPS_PROXY=http://127.0.0.1:7890
```

### 问题 3: Claude Code 无响应

**检查清单**:
- [ ] Proxy 服务正在运行
- [ ] `/health` 端点可访问
- [ ] CC Switch 配置正确
- [ ] 已重启 Claude Code 终端
- [ ] Base URL 没有多余的路径

**调试**:
```bash
# 查看 Proxy 日志
# 日志会显示是否收到请求

# 启用 DEBUG 模式
# 编辑 .env: LOG_LEVEL=DEBUG
```

### 问题 4: 缺少依赖

**现象**:
```
ModuleNotFoundError: No module named 'fastapi'
```

**解决**:
```bash
pip install -r requirements.txt
```

## 🔒 安全提示

1. **本地监听**: `PROXY_HOST=127.0.0.1` (不要改成 `0.0.0.0`)
2. **保护 API Key**: 不要提交 `.env` 到 Git
3. **日志脱敏**: 不记录完整 API Key
4. **生产部署**: 使用 HTTPS (Nginx 反代)
5. **访问控制**: 添加认证中间件(如需公开访问)

## 📈 性能优化

### 并发优化

根据负载调整 `MAX_AGENTS`:

| 并发请求 | 推荐 MAX_AGENTS |
|----------|-----------------|
| < 5 | 3 |
| 5-10 | 5 |
| 10-20 | 10 |
| > 20 | 考虑负载均衡 |

### 内存优化

- 流式响应: 减少内存占用
- 日志级别: 生产环境使用 INFO
- 队列限制: 避免任务堆积

### 网络优化

- 使用代理: 减少延迟
- 连接复用: httpx AsyncClient
- 超时控制: 避免长时间等待
